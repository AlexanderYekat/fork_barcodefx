Все шаблоны по умолчанию следует сохранять в папку templates.
Все шаблоны должны иметь расширение html.
Шаблон должен быть составлен с учетом html разметки.
Можно исползовать CSS стили.
Программа обрабатывает наиболее распространенные html теги, корректная обработка любых тегов не гарантируется.

Специализированные теги:

<gtin/> - при формировании pdf будет заменен на gtin.
С помощью атрибута delete_ai можно удалить идентификатор применения. По умолчанию false.

<serial/> - при формировании pdf будет заменен на серийный номер.
С помощью атрибута delete_ai можно удалить идентификатор применения. По умолчанию false.

<user_label_text/> - при формировании pdf будет заменен на значение из поля "текст на этикетке"

Для тегов gtin, serial, user_label_text можно задать дополнительные атрибуты string_start и string_end для вывода части текста.
Пример <serial string_start="0" string_end="2"/> выведет только первые два знака серийного номера.

<page_number/> - при формировании pdf будет заменен на порядковый номер этикетки.

<total_page_number/> - при формировании pdf будет заменен на общее количество этикеток.

<datamatrix height="62" width="62"/> - при формировании pdf будет заменен на датаматрикс. Требуется указывать атрибуты height и width - ширина и высота датаматрикса в pt.
С помощью атрибута delete_fnc можно отключить печать FNC1. По умолчанию false.
Опциональные атрибуты margin_right, margin_left, margin_top, margin_bottom для управления отступами от датаматрикса.
Опциональный атрибут - enconodation.
Возможные варианты:
DM_AUTO - the best encodation will be used
DM_ASCII - ASCII encodation
DM_C40 - C40 encodation
DM_TEXT - TEXT encodation (по умолчанию)
DM_B256 - binary encodation
DM_X12 - X12 encodation


В случае печати нескольких кодов на одной этикетке в настройках печати нужно указать сколько кодов печатать на этикетке (DATAMATRIX_PER_TEMPLATE_LABEL).
В шаблоне у спец тегов указывать какую по счету этикетку печатать через атрибут km_number. Нумерация начинается с нуля.


Теги каталога товаров:

В качестве имени тега используется значения первой строки каталога. Имя регистронезависимо.
За gtin принимается значение V_PROD_COVER_GTIN.
К тегу можно добавить атрибут type - тип тега.

Возможные значения:

STRING - печатает значение без изменения. Используется по умолчанию.

DATE - обработать как дату. Можно дополнить атрибутом date_format для изменения формата даты. По умолчанию "dd.MM.yyyy"

DICT - обработать как элемент справочника. Можно дополнить атрибутом use_key. Если true, то будет печатать ключ справочника, если false - значение справочника. По умолчанию - false.

EAN-13 - обработывается как штрихкод формата EAN-13. Атрибуты height и width обязательны. Можно дополнить атрибутом height, width, margin_right, margin_left, margin_bottom, margin_top, rotation. rotation позволяет воернуть одномерный штрихкод. Задавать в радианах. 90 градусов - 1,57 радианы.

BARCODE-128 - обработывается как штрихкод формата Code 128. Атрибуты height и width обязательны. Можно дополнить атрибутом height, width, margin_right, margin_left, margin_bottom, margin_top, rotation. rotation позволяет воернуть одномерный штрихкод. Задавать в радианах. 90 градусов - 1,57 радианы.

Пример <v_web_90001690 type="EAN-13" height="62" width="62" rotation="1.57"/>. В этом случае программа возмет значение из столбца v_web_90001690 и преобразует его в одномерный штрих код EAN-13.
